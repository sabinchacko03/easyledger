# Project: UAE Peppol-Ready Receipt & Credit Note System
# Stack: Laravel (Backend API) + React Native (Frontend)
$ DB: MySQL 8.0 (UTF-8MB4 for Arabic).
# Environment: Laravel 13, PHP 8.3+, Docker-based (Laravel Sail). Optimized for Hostinger KVM VPS deploymen
# Tech: Laravel 13 AI SDK, JSON:API Resources, Spatie Permission v7+
# App:
    - This is a simple app to generate a general purpose receipt by salesmen at collection points, for now
    - Our users are the supplier companies which are signing up through the web portal
    - The supplies will then add salesmen and allocate them a credentials
    - sales person will use the credential in the mobile app to login and generate the receipts
    - Receipts are general purpose for now, with a description as well.
    - We will later modify this app to be a tax ready system for the users to submit for asps
    - Receipts are sent to the customer companies and supplier companies.
    - There should be offline option in the phone to save the receipt info and once the internet is connected the recepits should be generated and sent.

## 1. Database & Models
- users Table
  - id, name, email, password
  - company_id (Nullable: Null for a new user, filled once they create/join a company)
  - role (e.g., admin or salesperson)
  - is_active: Boolean (to instantly revoke access if a salesman leaves the company).

- Table: salesperson_profiles (Optional Extended Info)
  - user_id (FK to users).
  - employee_id: String (Internal company ID).
  - daily_collection_limit: Decimal (A safety feature to alert the office if a salesman collects more than a certain amount in a day).

- Create 'tenants' (The App Owners) table with: name, logo, trn (15 digits), tin (10 digits), address, city, emirate (Enum: AUH, DXB, SHJ, AJM, UAQ, RAK, FUJ), email,owner_id (Foreign key to the User who created it)
- Create 'Customers' (The Collection Points) table
 - id, tenant_id (FK), name, trn, phone, email, address.
 - current_balance (Decimal 15,2).
- Create 'documents' table (The Ledger):
 - id, uuid, tenant_id, customer_id, salesperson_id.
 - type: enum (380=Receipt, 381=Credit Note).
 - doc_number: Unique sequential string per tenant (e.g., PV-2026-0001).
 - parent_id: self-referencing (Required for Credit Notes).
 - amount, currency (Fixed 'AED'), description.
 - payment_mode (Cash, Cheque, Bank Transfer).
 - evidence_image_path, gps_lat, gps_long.
 - amount_words_en, amount_words_ar (Tafqeet).
 - status: draft, synced, archived.

## Phase 1: Laravel 13 API
Multi-Tenancy: Implement a TenantScope trait to automatically filter all queries for customers and documents by tenant_id.
Tafqeet Helper: Create a utility to convert currency values into Arabic words (e.g., "فقط خمسة آلاف درهم لا غير").
Bilingual PDF: Setup a PDF generator using Tailwind-based HTML. Ensure Right-to-Left (RTL) support for Arabic fields using a font like 'Cairo'.
Audit Job: On document sync, trigger a background job to email the PDF to the customer and a BCC to the tenant's office.

## 5. React Native / Expo Specifications

- Enable 'New Architecture' in app.json (should be default in SDK 55).
- Use **Expo Router v7** for file-based navigation.

## 2. Credential Creation Workflow
Since you want the Supplier (Tenant Admin) to create these credentials, the workflow in your Laravel Web Portal should look like this:
Step | Action | Logic / Security
1. Admin Input| "Admin enters Name, Email (or Username), and a temporary password." | "Use Laravel 13 AI SDK to suggest ""Secure Passwords"" or auto-generate them."
2. Scoping | System automatically attaches the tenant_id of the logged-in Admin to the new user. | This ensures the salesman cannot see data from other suppliers.
3. Permission Assignment | Assign the salesperson role via Spatie. | Restricts them to only the Mobile API endpoints.
4. Communication | "System generates a ""Welcome Kit"" (PDF or Email) with a QR Code for easy login." | The salesman scans the QR in the React Native app to auto-fill the Tenant ID and Username.

## Salesperson Management Logic
- **User Creation:** Only users with 'Admin' role can create users with 'Salesperson' role.
- **Tenant Enforcement:** The `tenant_id` must be inherited from the creator. It cannot be manually selected.
- **Authentication:** Use **Laravel Sanctum**. Mobile app must send the `tenant_id` or sub-domain in the header to ensure correct scoping.
- **API Security:** - Salesmen can only `GET` customers belonging to their `tenant_id`.
    - Salesmen can only `POST` documents. They cannot `PUT` (edit) or `DELETE` once a document is synced (Anti-fraud).
